# Sentence Patterns

---

# The Building Blocks of English

Almost every English sentence is built using a small number of patterns.

Learning these patterns is much easier than memorizing grammar rules.

Once you recognize them, building sentences becomes natural.

---

# Pattern 1 — Someone Exists

Use this pattern to describe someone or something.

```
WHO

↓

BE

↓

DESCRIPTION
```

Examples

```
The server is online.

The application is available.

The engineer is busy.

The report is ready.

The customer is happy.
```

---

# Pattern 2 — Someone Acts

Use this pattern when an action does not affect another object.

```
WHO

↓

ACTION
```

Examples

```
The server restarted.

The customer arrived.

The deployment finished.

The application crashed.

The engineer smiled.
```

---

# Pattern 3 — Someone Affects Something

This is probably the most common pattern in English.

```
WHO

↓

ACTION

↓

TARGET
```

Examples

```
The engineer restarted the server.

The customer opened a ticket.

The application stored the data.

The deployment created an incident.

The script generated a report.
```

---

# Pattern 4 — Something Exists

Sometimes there is no important actor.

English simply introduces existence.

```
THERE IS

THERE ARE
```

Examples

```
There is a problem.

There is enough information.

There are three servers.

There are many users.

There are a few incidents.
```

---

# Pattern 5 — Someone Gives Something

Some actions involve three participants.

```
WHO

↓

ACTION

↓

WHAT

↓

TO WHOM
```

Examples

```
The engineer sent the report to the customer.

She gave the server a new name.

The application returned an error to the user.

The teacher showed the students a diagram.

The manager assigned a task to the team.
```

---

# Recognize Before Building

When reading English, ask yourself:

```
Which pattern is this?
```

When speaking English, ask yourself:

```
Which pattern do I need?
```

Once you recognize the pattern, the sentence becomes much easier to build.

---

# Engine Insight

Native speakers do not memorize thousands of sentence structures.

They repeatedly use a small number of patterns.

Master these patterns first.

Everything else grows from them.
