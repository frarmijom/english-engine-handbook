# Building an Action

Engine 02 taught us how to build an idea.

```
WHO

↓

ACTION

↓

TARGET

↓

DETAILS
```

Engine 03 answers a different question.

**How does the Action change while the idea remains the same?**

---

# The Skeleton Never Changes

No matter where the action lives, the structure of the idea remains stable.

```
WHO

↓

ACTION

↓

TARGET

↓

DETAILS
```

Only the Action changes its form.

---

# Structure 01 — Simple Action

```
WHO

+

ACTION

+

TARGET

+

DETAILS
```

Examples

```
I work every day.

She studies English.

The engineer restarted the server.

The deployment finished yesterday.
```

---

# Structure 02 — Action in Progress

```
WHO

+

BE

+

ACTION-ing

+

TARGET

+

DETAILS
```

Examples

```
I am working.

She is studying English.

The engineer is restarting the server.

The deployment is running.
```

+

+

DETAILS

```
WHO

+

+

ACTION

+

TARGET

+

DETAILS
```

Examples

```

```

---

# Structure 05 — Possible Action

```
WHO

+

WOULD / COULD / MIGHT / SHOULD

+

ACTION

+

TARGET

+

DETAILS
```

Examples

```
I would work there.

She could study tonight.

The engineer might restart the server.

We should investigate the issue.
```

---

# Structure 06 — Connected Actions

Some ideas contain two actions.

```
ACTION A

↓

ACTION B
```

The second action depends on the first.

---

## Future Condition

```
IF

+

ACTION

↓

WILL

+

ACTION
```

Example

```
If I buy bread today,

I will eat toast tomorrow.
```

---

## Imagined Situation

```
IF

+

ACTION

↓

WOULD

+

ACTION
```

Example

```
If I had more time,

I would study English every day.
```

---

## Imagined Past

```
IF

+

PAST ACTION

↓

WOULD HAVE

+

ACTION
```

Example

```
If I had bought bread,

I would have eaten toast.
```

# One Skeleton

Observe these sentences.

```
I work every day.

I worked yesterday.

I am working now.

I will work tomorrow.

I would work if I had time.
```

The idea never changes.

```
WHO

↓

ACTION

↓

DETAILS
```

Only the Action changes its structure.
